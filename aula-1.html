Requisitos de sistemas de jogos digitais

De acordo com o dicionário Aurélio, requisito é a condição básica e necessária para se obter alguma coisa ou para alcançar determinado propósito.

Durante o processo de desenvolvimento de um sistema de software é necessário que alguns aspectos importantes estejam definidos de tal forma que, o resultado almejado e também proposto, consiga ser alcançado de maneira satisfatória.

Esses aspectos são inerentes a definição e elicitação de requisitos. As etapas de elicitação de requisitos é muito importante para o desenvolvimento de um projeto de software. Elas exigem uma descrição passo a passo do que acontecerá com cada ação do usuário. Dessa forma, o desenvolvimento será mais confiante e os resultados estarão alinhados às expectativas do cliente. A falha dos requisitos básicos do software pode interromper todo o processo e criar alguns problemas, como erro de execução, solicitações desnecessárias, perda de tempo no desenvolvimento.

A falha dos requisitos básicos do software pode interromper todo o processo e criar os seguintes problemas: difícil de usar; erro de execução; desorganizado; solicitações desnecessárias; falta de funções importantes; perda de tempo de desenvolvimento. Toda essa situação pode ser frustrante para clientes e desenvolvedores. Os orçamentos podem precisar ser revisados, afetando os prazos de entrega. Portanto, quanto mais coesa e clara a especificação dos requisitos de software, melhor para o sucesso desse sistema.

Definir requisitos é a primeira fase no ciclo de desenvolvimento do projeto.  A partir dela serão definidos vários serviços, funcionalidades, processos, bem como possíveis restrições que o sistema que o software terá.

Requisitos de Hardware

Você já tentou instalar um jogo no seu computador, mas não conseguiu devido a incompatibilidade da sua máquina com as exigências mínimas para rodar o jogo?

Pois bem esse exemplo retrata sobre os requisitos de hardware, que referem-se às recomendações e características consideradas  mínimas onde o dispositivo em que o sistema será implementado deve possuir para que o mesmo funcione perfeitamente.

Os requisitos mínimos de hardware são as especificações técnicas mínimas necessárias para que um jogo possa ser executado em um dispositivo, porém com desempenho básico e gráficos em níveis mais baixos. Já os requisitos recomendados indicam a configuração ideal para uma experiência de jogo mais suave, com desempenho otimizado e gráficos de alta qualidade.

Exemplificando, pode-se citar os requisitos mínimos de hardware que um computador deverá ter para a instalação de um jogo digital como: processador, placa-mãe, memória RAM, placa de vídeo, unidade de armazenamento(HDD/SSD).

Os requisitos mínimos de hardware indicam a configuração mínima necessária para rodar o jogo, com desempenho básico e gráficos em níveis mais baixos, enquanto os requisitos recomendados indicam a configuração ideal para uma experiência de jogo mais suave, com desempenho otimizado e gráficos de alta qualidade.

É importante verificar os requisitos de hardware de um jogo antes de comprá-lo ou instalá-lo em seu dispositivo, para garantir que seu dispositivo atenda aos requisitos mínimos e recomendados. Caso contrário, o jogo pode não funcionar adequadamente ou pode ter um desempenho abaixo do esperado.

Se o dispositivo não atender aos requisitos mínimos, o jogo pode não funcionar adequadamente ou pode ter um desempenho abaixo do esperado, com problemas como lentidão, travamentos, ou até mesmo não funcionar. Por outro lado, se o dispositivo atender apenas aos requisitos mínimos, o desempenho do jogo pode ser comprometido, com baixa qualidade gráfica e desempenho insatisfatório.

Para uma experiência de jogo ideal, é recomendável que o dispositivo atenda aos requisitos recomendados, para aproveitar todas as funcionalidades e recursos do jogo de forma adequada. Verificar os requisitos de hardware é uma prática importante para garantir uma experiência de jogo satisfatória.

Requisitos de Software

De forma prática, requisitos de softwares é a etapa relacionada a entender as necessidades do cliente e traduzi-las para o sistema, ou seja, o software. Isso é extremamente importante para que a equipe de desenvolvimento consiga chegar mais próximo do produto final evitando, assim, retrabalho.

Requisitos referem-se à definição de atributos, funcionalidades, características ou algo que o sistema deva restringir e tornar-se útil para seu usuário final.

De forma mais sucinta, um requisito é algo que o sistema deve fazer (funções) ou algo que o restrinja. Os requisitos de softwares são classificados em requisitos não funcionais e requisitos funcionais.

Colher requisitos não é um processo simples e deve ser feito com cautela.

Os passos a seguir mostram o que deve ser seguido para definir os requisitos de software de um projeto.

  • Discutir o projeto com os interessados: Conhecer clientes, usuários finais e até patrocinadores. Conversar com eles sobre a expectativa do projeto. E deixar claro que, nem todos os requisitos citados nas conversas, são viáveis para o projeto.
  • Definir o público: Saber exatamente quem vai usar o software para que ele seja adaptado a essas pessoas.
  • Determinar o escopo do projeto: determinar e definir quais elementos estarão presentes e envolvidos no projeto.
  • Manter as coisas simples: os requisitos devem ser sempre objetivos, claros e diretos mesmo que o projeto seja complexo.
  • Usar linguagem clara e precisa: ser o mais preciso possível ajuda a evitar contratempos ao longo do projeto. A equipe de desenvolvimento deve ter um bom entendimento de todos os requisitos.
  • Acompanhar os requisitos: todos os requisitos devem ser acompanhados de maneira cuidadosa para que atendam as necessidades do cliente e usuário. Evita retrabalho, pois é possível corrigir erros o mais rápido possível.
  • Documentar todas as alterações: à medida que o projeto evolui os requisitos de software podem evoluir. Então documentá-los é tão importante. É preciso que a equipe de desenvolvimento tenha ciência que nem tudo sairá como planejado.
  • Delimitar uma conexão entre cada requisito e a finalidade do software: é preciso e necessário que cada requisito esteja associado ao principal objetivo do sistema que está em desenvolvimento. Isso faz com que o cliente fique satisfeito com o produto final.
  • Criar documentação de especificação de requisitos: o documento de requisitos é fundamental no desenvolvimento de um sistema. Ele é elaborado para servir como resumo do projeto, contendo pontos essenciais como: escopo do projeto, as definições necessárias e personas. Além disso, os requisitos funcionais e não funcionais.

    • Os requisitos de softwares são classificados em requisitos não funcionais e requisitos funcionais.

    Requisitos funcionais

    Os requisitos funcionais devem descrever de uma maneira clara e explícita todas as funcionalidades que o sistema contém.

  • É necessário que os requisitos funcionais possuam atributos que o identifiquem;
  • A completude é um atributo de um requisito funcional, que se refere a definição de todos os serviços do sistema;
  • O atributo de consistência diz respeito a não contradição das definições do sistema.

    • A seguir separamos alguns exemplos de requisitos funcionais no desenvolvimento de um jogo digital:

  • Opção de escolha do idioma, disponível na tela de Opções(Options) antes do jogo iniciar.
  • Permissão ao usuário escolher os níveis de dificuldade “fácil”, “normal” e “difícil” através da tela de Opções antes de o jogo iniciar.
  • Configurações do áudio utilizando recursos de teclas de “ativar” ou “desativar” o som através da tela de Opções.

    • Um requisito funcional não é uma funcionalidade, mas uma necessidade funcional, ou seja, uma função que o software deve atender. Nisso, uma funcionalidade pode requerer um ou mais requisitos funcionais.

    Requisitos não funcionais

    Os requisitos não funcionais são aqueles que descrevem restrições ao sistema e aspectos de qualidade, considerados importantes, e que não estão relacionados à funcionalidade.

    Em um processo de desenvolvimento de um jogo digital, os requisitos não funcionais não irão descrever o que o sistema fará, mas sim, como será feito. Eles estabelecem como o sistema de um jogo irá se comportar em determinadas situações.

    Um exemplo de um requisito não funcional de um jogo digital é sobre qual plataforma o jogo será desenvolvido.

    O requisito não funcional possui categorias imprescindíveis de conhecê-las para haver uma melhor organização do projeto.
